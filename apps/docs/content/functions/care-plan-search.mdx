title: "carePlan.search"

## Request Parameters

### Native Request Elements
| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|-----------|
| category (String) | Type of plan. 38717003 should be used to search for longitudinal care plans. In addition, category type "assess-plan" is always included in search results. | false | false |
| subject (String) | Patient FHIR resource ID for the patient associated with the care plan. | conditional | false |

### Post-filter Parameters
| Name | Description | Is Optional | Array |
|------|-------------|----------|--------|
| activity-code (String) | Order or procedure code. | true | false |
| activity-date (String) | Specified date occurs within period | true | false |
| condition (String) | The list of health issues this longitudinal care plan addresses. | true | false |
| goal (String) | Goals for this longitudinal care plan. | true | false |
| instantiates-canonical (String) | Unsupported | true | false |
| intent (String) | For longitudinal care plans, this always equals "plan". | true | false |
| patient (String) | Patient FHIR resource ID for the patient associated with the care plan. | conditional | false |
| status (String) | Always "active." | true | false |
