# Allergies

## Request Parameters

### Native Parameters
| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|----------|
| code | Must be "allergies" | false | false |
| patient | The patient with whom the list is associated | conditional | false |
| subject | The patient with whom the list is associated | conditional | false |

### Post-filter Parameters
| Name | Description | Required | Array |
|------|-------------|----------|--------|
| empty-reason | If the patient has no allergies documented, the API returns an empty reason of "notstarted". | Yes | No |
| item | AllergyIntolerance FHIR ID | Yes | No |
| status | Status of the allergy list. Always "current" because each list represents only the patient's current allergies. | Yes | No |
| title | The title representing this entity. Always "Allergy List". | Yes | No |

# Medications

## Request Parameters

### Native Request Elements
| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|----------|
| code | The code representing the type of list to search. Must be "medications" | conditional | false |
| encounter | Reference to the Encounter resource with which the List is associated | conditional | false |
| patient | The Patient resource with whom the List is associated | conditional | false |
| subject | Reference to the Patient resource with whom the List is associated | conditional | false |


### Post-filter Parameters
| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|-----------|
| date | Unsupported | true | false |
| emptyReason | The reason why the list is empty. Only populated when the list is empty, and preferred to be one of the values here.\nWill be "nilknown" when the med list has been marked as reviewed for the encounter.\nWill be "notasked" for all other encounters.\nWill be "withheld" when in a MyChart context and all meds have been filtered.\nWill be "unavailable" when we recorded a message in the "note" property above and quit before searching for meds. | true | false |
| identifier | Unsupported | true | false |
| item | Populated with a reference to the MedicationRequest resource for the represented order. | true | false |
| notes | The annotation - text content | true | false |
| source | Unsupported | true | false |
| status | Status of the medication list. Always "retired" because each list only represents an encounter's changes. Values can be found here. | true | false |
| title | Unsupported | true | false |
