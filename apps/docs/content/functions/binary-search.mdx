## Clinical Notes
This API takes in a Binary FHIR ID and returns an HTML version of the note. 

### Native Request Elements
To get the Binary FHIR ID, you should use [DocumentReference.Search](/functions/document-reference-search) to obtain the FHIR ID.

| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|-----------|
| id (String) | The FHIR ID. | false | false |

### Native Response Elements

| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|-----------|
| Binary (Binary) | A single instance of a FHIR Binary resource. | false | false |

### Types

#### Binary

| Name | Description | Is Optional | Is Array |
|------|-------------|-------------|-----------|
| contentType (String) | Mime content type string. This is "text/html" or "text/rtf" for clinical notes. | false | false |
| data (String) | The note or document content as a base64-encoded string. | false | false |

### Example Payload
```
fhir_id = client.documentReference.search()[][]
client.binary.read('example')
```
